# CypherPilot — Non-Goals

> Explicitly documenting what we are **not** building is as important as documenting what we are building. These boundaries prevent scope creep and keep the MVP focused.

---

## Out of Scope for MVP (v0.x)

### Authentication & Multi-User
- No user registration, login, or session management
- No role-based access control (RBAC)
- No organizations, teams, or workspace sharing
- No SaaS multi-tenancy

### Reason
Single-user, self-hosted desktop tool for MVP. Auth adds complexity (JWT, password hashing, session storage, CSRF, rate limiting) that distracts from the core value proposition. The architecture will use a request-scoped session model that can be retrofitted with auth later.

---

### CLI Interface
- No command-line tool for test generation or analysis
- No programmatic API for headless/CI usage (though the FastAPI backend exposes REST APIs that could serve this purpose)

### Reason
A CLI doubles the surface area (argument parsing, output formatting, exit codes, config file handling) without enough marginal value for MVP. The REST API + web UI is sufficient. A CLI can be added as a separate consumer of the same API.

---

### Test Execution
- CypherPilot does **not** run tests
- CypherPilot does **not** integrate with CI pipelines to trigger test runs
- CypherPilot does **not** collect test results from external runners

### Reason
We generate test cases and analyze failures — we do not execute or orchestrate test runs. Execution is the domain of existing tools (PyTest, Playwright CLI, CI platforms). Adding execution would require test environments, dependency management, output capture, timeouts, and resource isolation — a massive expansion of scope.

---

### Real-time/Collaborative Features
- No WebSockets, SSE, or real-time updates
- No collaborative editing or shared sessions
- No notifications

### Reason
Stateless HTTP request-response is simpler, more reliable, and easier to test. Real-time features can be added later when collaboration becomes a priority.

---

### AI Response Caching
- No caching of AI responses (no Redis, no database cache)
- No deduplication of identical requests

### Reason
Token costs for MVP are manageable with free-tier providers (Gemini). Caching is a premature optimization that adds cache invalidation complexity. The database schema will have a `cache_key` nullable column for future use.

---

### Advanced Analytics & Reporting
- No dashboards with charts or metrics
- No token usage analytics beyond raw metadata
- No export to third-party test management tools (TestRail, Xray, Zephyr)
- No Allure report ingestion (deferred to Future Phases)

### Reason
Analytics are valuable but non-essential for MVP. Raw metadata (provider, model, latency, tokens) will be stored from day one, but no visualization layer.

---

### Prompt Playground
- No UI for editing or testing prompts interactively
- No prompt version comparison

### Reason
Prompt templates are versioned Markdown files managed via git. A playground is a future enhancement once the prompt architecture is proven in production.

---

### Visual/Screenshot Analysis (Advanced)
- Screenshot analysis for failure diagnostics is a stretch goal within v0.4.0
- Text-based log analysis is the primary failure analysis mode

### Reason
Vision model analysis adds significant complexity (image upload, storage, multimodal prompt design, cost). The MVP failure analysis module should work well with text inputs first; screenshot analysis is a documented enhancement.

---

### Internationalization (i18n)
- English-only for MVP
- No translation infrastructure

### Reason
Adding i18n before the product-market fit is established is premature. All prompt templates are in Markdown and can be localized later by creating language-specific template variants.

---

## Explicitly Out of Scope for All Phases (unless revised)

- **Test recorder / codegen** (like Playwright Codegen) — CypherPilot generates test *design* from requirements, not test *scripts* from browser interactions
- **Browser automation** — CypherPilot does not control browsers
- **Performance/load testing** — not a load testing tool
- **Security testing** — not a security scanner (though security test cases may be generated by Requirement Analysis)
- **Mobile testing** — no iOS/Android-specific support (though generated test methodology applies)
- **Low-code/no-code test builder** — outputs are code-first (Python, PyTest)
- **Marketplace / plugin system** — no third-party extension mechanism

---

## Why Non-Goals Matter

Every item on this list represents work we are **intentionally not doing**. This protects the MVP from:

- **Scope creep** — "Let's just add a quick auth screen" becomes a 3-week detour
- **Architectural bloat** — anticipatory complexity for features we may never build
- **Diffusion of focus** — doing fewer things better beats doing many things poorly

When a reviewer asks "Why doesn't CypherPilot have X?", we have a documented, defensible answer.
